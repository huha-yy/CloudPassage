# 简历项目描述

## 项目名称

多 Agent 智能图文创作平台

## 项目定位

基于 `Spring AI Alibaba + Spring Boot + Vue 3 + TypeScript + Redis + MySQL + Docker` 构建的 AI 全栈项目，面向文章生成、配图生成、任务恢复与 Agent 调试场景。项目不仅实现了多阶段内容创作，还重点增强了 Agent 工程化能力，包括任务记忆、节点观测、单节点重放、失败恢复和长期记忆上下文。

## 适合简历上的一句话版本

设计并开发基于 Spring AI Alibaba 的多 Agent 智能图文创作平台，支持文章生成、图片策略路由、正文评审、任务记忆、节点重放与可观测调试，提升 AI 工作流的可恢复性、可解释性和工程可维护性。

## 技术栈

- 后端：`Java 21/23`、`Spring Boot`、`Spring AI Alibaba`
- 前端：`Vue 3`、`TypeScript`、`Vite`
- 存储：`MySQL`、`Redis`
- 部署：`Docker Compose`
- 能力组件：`SSE`、`多 Agent 编排`、`任务快照`、`节点日志`、`长期记忆`

## 可直接放简历的项目描述

### 版本一：偏 AI 全栈开发

负责开发一套基于 `Spring AI Alibaba` 的多 Agent 智能图文创作平台，完成前后端联调、任务流设计与本地 Docker 化部署。后端围绕标题生成、大纲生成、正文生成、正文评审、图片策略路由、配图生成和图文合并构建完整 AI 工作流；前端实现文章创建、生成过程观测、任务调试与结果展示。项目进一步补齐了任务快照、失败恢复、单节点重放、任务记忆与长期记忆上下文等能力，使系统从“能调用大模型”升级为“可恢复、可调试、可迭代”的 Agent 工程项目。

### 版本二：偏 AI Agent 开发

基于 `Spring AI Alibaba` 设计并实现多 Agent 内容生成链路，将文章创作拆分为标题生成、结构生成、正文生成、内容评审、图片路由和配图生成等节点，并构建节点级执行观测、重放恢复和记忆增强机制。通过接入任务记忆、用户偏好记忆和长期记忆上下文，使历史成功/失败经验真实参与图片路由和正文 Reviewer 决策，提升 Agent 执行稳定性、可解释性和后续优化效率。

## 核心亮点

- 设计多 Agent 工作流，覆盖标题、大纲、正文、Reviewer、图片 Router、图片生成、图文合并等关键节点
- 基于 `SSE` 实现生成进度实时推送，支持前端实时查看执行状态
- 实现任务快照与阶段恢复机制，支持失败任务继续执行
- 实现单节点重放能力，可从正文、Reviewer、配图分析、图片生成、图文合并等节点继续运行
- 构建节点日志、节点时间线、调试面板，提升 Agent 调试与问题定位效率
- 设计任务记忆、用户偏好记忆、长期记忆上下文，并让记忆真实参与 Agent 决策
- 完成 Docker 化启动方案，支持本地一键构建和运行

## 可直接使用的简历要点

- 基于 `Spring AI Alibaba` 实现多 Agent 图文生成工作流，支持标题生成、大纲生成、正文生成、正文评审、图片路由与图文合并
- 设计任务快照、节点日志、SSE 实时推送与单节点重放机制，提升 AI 工作流的可恢复性和可观测性
- 构建任务记忆、用户偏好记忆和长期记忆上下文，并将历史经验接入图片 Router 与正文 Reviewer，增强 Agent 决策能力
- 使用 `Vue 3 + TypeScript` 搭建任务调试台，支持节点时间线、执行摘要、失败原因与重试入口展示
- 基于 `MySQL + Redis + Docker Compose` 完成本地部署与运行链路，支撑完整业务验证
