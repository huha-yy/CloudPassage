# 第三阶段实施方案

## 阶段目标

第三阶段聚焦“Agent 能力增强”，目标不是再做一轮大范围页面改造，而是在第二阶段已经稳定的执行骨架上，补足更像 Agent 系统的能力：记忆、路由、评审、工具策略与更细粒度恢复。

本阶段完成后，希望项目从“可观测工作流”继续升级为“可规划、可反馈、可迭代”的多 Agent 系统。

## 第三阶段重点方向

### 1. 记忆架构落地

优先补两层记忆：

- 任务级记忆：保存一次创作过程中的阶段结果、关键决策、失败节点、图片策略
- 用户级偏好记忆：保存用户常用风格、常选配图方式、标题偏好、常见修改倾向

本阶段不急着做复杂向量检索，先把“结构化短期记忆 + 用户偏好记忆”做稳。

### 2. 引入 Planner / Router 能力

在现有固定流程上增加轻量路由判断，例如：

- 当前任务是否需要图片
- 当前任务更适合哪种图像工具
- 当前任务是否需要 Mermaid / SVG / 插图混合策略
- 当前任务是否需要更长推理或更严格结构化输出

目标是让系统开始具备“根据任务差异动态选择执行策略”的能力。

当前优先落地的第一个真实决策点是“图片策略 Router”：

- 根据标题 / 大纲 / 用户偏好 / 失败记忆判断本次是否需要图片
- 在用户显式选择、用户历史偏好、默认策略之间做有优先级的决策
- 优先区分“图解型内容”和“写实配图型内容”，决定更偏向 `MERMAID` / `SVG_DIAGRAM` 还是 `PEXELS`
- 把决策结果同步写入任务记忆、节点观测日志、节点回放快照可见链路

### 3. 增加 Reviewer / Refiner 闭环

为标题、大纲、正文增加基础评审与修正能力：

- 标题候选质量检查
- 大纲结构完整性检查
- 正文是否偏题、重复、断裂
- 图片需求是否与段落内容一致

本阶段建议先做轻量评审，不追求复杂多轮自博弈。

### 4. 升级工具路由与降级策略

围绕图片与图表工具增强：

- 明确 Pexels、Mermaid、SVG、AI 生图等工具的适用条件
- 工具失败后有清晰降级逻辑
- 记录工具命中率、失败率、回退路径

目标是让工具调用从“能用”变成“可选、可退、可分析”。

### 5. 推进真正单节点重放

在第二阶段“节点触发、阶段续跑”的基础上，继续往真正 node-level replay 推进：

- 单独重试图片生成节点
- 单独重试图文合成节点
- 单独重试某个工具调用节点

这部分是第三阶段里最有价值的执行引擎增强项之一。

## 建议拆分顺序

### 第一小步：记忆底座

- 定义任务记忆结构
- 定义用户偏好结构
- 明确保存时机与读取时机

### 第二小步：路由与评审

- 增加 Planner / Router
- 增加 Reviewer / Refiner
- 明确这些节点在主链路中的插入位置

当前拆分顺序调整为：

1. 先落地图片策略 Router（规则法，非 LLM 路由）
2. 再补标题 / 大纲 / 正文 Reviewer
3. 最后再决定是否引入更复杂的 Planner

### 第三小步：节点级重放与工具策略

- 抽离真正可单独重试的节点
- 让重试不再总是回退整个阶段
- 统一工具调用决策与失败降级记录

## 预期接口与模型变化

第三阶段可能新增或扩展以下内容：

- 任务记忆 VO / 实体 / 缓存模型
- 用户偏好配置模型
- Planner / Reviewer 相关事件类型
- 更细粒度节点状态与恢复原因字段
- 工具调用观测字段（工具名、降级来源、失败类型）

## 验收标准

### 后端验收

- 至少一类记忆能力可以真实影响执行结果
- 至少一个路由决策点不再是写死逻辑
- 至少一个评审节点能对生成结果进行自动修正或拦截
- 至少一个失败节点可实现真正单节点级重放

当前已完成：

- 任务级记忆
- 用户级偏好记忆
- 节点调试面板与节点回放查询链路
- Prompt / 模型参数配置化底座

当前正在推进：

- 图片策略 Router 第一版

### 前端验收

- 能看到新增的路由 / 评审 / 重试结果提示
- 恢复与重试链路不因为新增 Agent 能力被破坏
- 执行观测字段仍然保持一致语义

## 本阶段边界

- 不做大规模 UI 重设计
- 不同时改 Java / Go / Python 三套后端
- 不一次性引入复杂 RAG 平台或多模型调度平台
- 不为了“像论文系统”而过早增加难维护的复杂模块

## 阶段结论

第三阶段不是简单继续加功能，而是让项目真正体现“Agent 系统”的核心价值：会记忆、会判断、会自检、会恢复、会根据任务差异调整策略。完成后，项目会更像一套可以持续演进的智能创作平台，而不只是固定流程生成器。
