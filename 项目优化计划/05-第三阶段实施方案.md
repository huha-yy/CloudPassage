# 第三阶段实施方案

## 阶段目标

第三阶段聚焦“Agent 能力增强”。目标不是继续做大范围页面改造，而是在第二阶段已经稳定的执行骨架上，补足更像 Agent 系统的能力：记忆、路由、评审、工具策略与更细粒度恢复。

这一阶段完成后，项目应从“可观测工作流”继续升级为“可规划、可反馈、可迭代”的多 Agent 系统。

## 第三阶段重点方向

### 1. 记忆架构落地

- 任务级记忆：保存一次创作过程中的阶段结果、关键决策、失败节点、图片策略、评审结论
- 用户级偏好记忆：保存常用风格、配图方式、失败偏好与修订倾向

当前不急着引入复杂向量检索，先把“结构化短期记忆 + 用户偏好记忆”做稳。

### 2. 引入 Planner / Router 能力

先从规则路由做起，让系统具备按任务差异动态选择执行策略的能力。

当前已优先落地的是真实决策点：

- 图片策略 Router
- 图片失败降级 / 重试 Router

已实现目标：

- 基于标题 / 大纲 / 用户偏好 / 任务记忆选择图片方法
- 失败后形成候选链路与最终落点
- 决策结果进入任务记忆、节点日志、节点回放快照、前端调试面板

### 3. 增加 Reviewer / Refiner 闭环

先做轻量评审，不做复杂多轮自博弈。

当前已落地：

- 正文 Reviewer 第一版
- 插入位置：`agent3_generate_content` 后、`agent4_analyze_image_requirements` 前
- 覆盖范围：偏题、重复、衔接、结尾收束
- 输出进入任务记忆、节点日志、节点回放快照

### 4. 工具路由与降级策略升级

围绕图片与图表工具增强：

- 明确 Pexels、Mermaid、SVG、AI 生图等工具适用条件
- 工具失败后有清晰降级逻辑
- 记录命中率、失败率、回退链路

当前这一块第一版已经完成，后续重点是继续做统计与策略迭代。

### 5. 推进真正的单节点重放

在第二阶段“节点触发、阶段续跑”的基础上，继续推进 node-level replay：

- 单独重试正文评审节点
- 单独重试图片生成节点
- 单独重试图文合成节点

这部分是第三阶段里最有价值的执行引擎增强之一。

## 当前拆分顺序

1. 图片策略 Router
2. 图片失败降级 / 重试 Router
3. 正文 Reviewer 第一版
4. 节点调试台增强
5. 单节点重放继续扩面验证

## 验收标准

### 后端验收

- 至少一类记忆能力真实影响执行结果
- 至少一个路由决策点不再是写死逻辑
- 至少一个评审节点能对生成结果做自动修正或拦截
- 至少一个失败节点可实现真正单节点级重放

### 前端验收

- 能看到新增的路由 / 评审 / 降级 / 重试结果提示
- 恢复与重试链路不因新增 Agent 能力被破坏
- 执行观测字段仍保持一致语义

## 阶段结论

第三阶段不是简单继续加功能，而是让项目真正体现“Agent 系统”的核心价值：会记忆、会判断、会自检、会恢复、会根据任务差异调整策略。
