# Agent 升级阶段里程碑看板

## 使用说明

- 这份文档用于跟踪当前 Agent 能力升级的阶段进度。
- 每完成一个阶段或一个关键子项，就实时更新“当前状态 / 已完成 / 下一步”。
- 状态约定：
  - `已完成`
  - `进行中`
  - `未开始`

- 当前状态：`已完成`
- 目标：把核心 Agent 的 `prompt`、`model`、`temperature`、`maxTokens`、`topP` 从硬编码改成可配置、可记录。
- 已完成：
  - 4 个核心 Agent 接入配置化 profile
  - `promptKey / promptVersion / model / temperature / maxTokens / topP` 已进入节点观测链路
  - 前端执行观测面板已能展示这些配置命中信息
- 产出重点：
  - 后续调 prompt、切模型、做 A/B 对比不再需要直接改代码

### 阶段 2：节点级观测与调试能力
- 当前状态：`进行中`
- 目标：让多 Agent 工作流具备“可观察、可定位、可调试”的能力。

  - `/article/node-replay/{taskId}` 查询接口
  - 前端“节点调试面板”接入创建页与详情页
  - 面板筛选与“重试该节点”入口已打通创建页
- 当前做到：
  - 能看节点状态、耗时、模型参数、输入摘要、输出摘要、错误信息、是否可回放
  - 已补充图片降级信息与正文 Reviewer 决策信息展示
- 下一步：
  - 继续把前端面板升级成更完整的“节点调试台”
  - 增加选中节点详情区、关键摘要聚合、排查效率优化

### 阶段 3：真正的单节点重放执行
- 当前状态：`进行中`
- 目标：不是按 phase 重跑，而是按 node 快照真正继续执行。
- 当前做到：

  - `/article/retry-node` 已改成进入后端节点级重放入口，而不是直接回退到整段 `resumeTask`

  - 再把“节点重放”从内容阶段扩展到更多节点，或补更细粒度的保护策略

### 阶段 4：任务记忆架构升级
- 当前状态：`进行中`
- 目标：让记忆层从“摘要展示”升级成“可被 Agent 利用的上下文资产”。
- 计划内容：
  - 用户偏好记忆
  - 失败模式记忆
  - 重试策略记忆
  - 节点产物沉淀与复用
- 已完成：
  - 用户偏好记忆已接入任务完成 / 失败沉淀
  - 新建创作任务时已自动应用用户常用风格与常用配图方式
  - 前端创建页已能读取 `user-preference` 作为默认策略来源
  - 任务级记忆已接入图片策略、图片降级、正文评审三类关键信号
- 下一步：
  - 继续让记忆不只“展示”，而是真实参与后续 Agent 决策
  - 把图片策略 / Reviewer 结果继续沉淀成更稳定的策略信号

### 阶段 4.1：图片策略 Router 与失败降级
- 当前状态：`已完成`
- 目标：让图片工具调用从“静态命中一种方法”升级为“先路由、失败再降级、全程可观测”。
- 已完成：
  - 图片策略 Router 第一版接入主链路
  - 路由结果已展示到任务记忆面板与节点调试面板
  - 路由元数据已进入节点日志与节点回放快照
  - 图片失败降级 / 重试 Router 已落地
  - 已支持“原方法 -> 候选链 -> 最终方法 -> 原因 -> 最终结果”全链路可见
  - 降级信号已沉淀回任务记忆

### 阶段 4.2：正文 Reviewer / Refiner 第一版
- 当前状态：`已完成`
- 目标：让正文在进入图片分析前，先经过一次轻量质量检查与最小修订。
- 已完成：
  - 已明确插入位置：`agent3_generate_content` 之后、`agent4_analyze_image_requirements` 之前
  - 已明确第一版范围：偏题、重复、衔接、结尾收束
  - 已新增正文 Reviewer 节点
  - Reviewer 输出已进入任务记忆、节点日志、节点回放快照
  - 前端任务记忆面板已新增“正文评审”区块
  - 节点调试面板已能展示 Reviewer 决策来源、原因、摘要
- 当前结论：
  - 第一版已具备“轻量评审 + 最小修订 + 可观测 + 可沉淀”的闭环
- 下一步：
  - 是否继续演进为更强的 Refiner，多轮修订暂不急着做
