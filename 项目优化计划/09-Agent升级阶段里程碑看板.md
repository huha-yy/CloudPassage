# Agent 升级阶段里程碑看板

## 使用说明

- 这份文档用于跟踪当前 Agent 能力升级的阶段进度。
- 每完成一个阶段或一个关键子项，就实时更新“当前状态 / 已完成 / 下一步”。
- 状态约定：
  - `已完成`
  - `进行中`
  - `未开始`

## 总体阶段

### 阶段 1：Prompt / 模型 / 参数配置化

- 当前状态：`已完成`
- 目标：把核心 Agent 的 prompt、model、temperature、maxTokens、topP 从硬编码改成可配置、可记录。
- 已完成：
  - 4 个核心 Agent 接入配置化 profile
  - promptKey / promptVersion / model / temperature / maxTokens / topP 已进入节点观测链路
  - 前端执行观测面板已能展示这些配置命中信息
- 产出重点：
  - 后续调 prompt、切模型、做 A/B 对比不再只能改死代码

### 阶段 2：节点级观测与调试能力

- 当前状态：`进行中`
- 目标：让多 Agent 工作流具备“可观察、可定位、可调试”的能力。
- 已完成：
  - 节点级日志增强
  - 节点回放快照后端结构
  - `/article/node-replay/{taskId}` 查询接口
  - 前端“节点调试面板”接入创建页与详情页
  - 面板筛选与“重试该节点”入口已接通创建页
- 当前做到：
  - 能看节点状态、耗时、模型参数、输入摘要、输出摘要、错误信息、可回放状态
- 下一步：
  - 面板筛选：只看失败 / 只看可回放 / 按节点过滤
  - 进入真正的单节点重放执行

### 阶段 3：真正的单节点重放执行

- 当前状态：`进行中`
- 目标：不是按 phase 重跑，而是按 node 快照真正继续执行。
- 当前做到：
  - `ArticleWorkflowStateFactoryImpl` 已优先叠加 Redis 任务快照，节点重放可拿到失败当下的上下文
  - `/article/retry-node` 已改为进入后端节点级重放入口，而不是直接回退到整段 `resumeTask`
  - 内容阶段首版已支持从 `agent3_generate_content` / `agent4_analyze_image_requirements` / `agent5_generate_images` / `agent6_merge_content` 开始继续执行
  - 节点重放过程中继续复用现有 SSE、节点日志、节点快照、任务记忆与完成态落库链路
- 下一步：
  - 补一轮真实任务验证，重点确认图片节点重放后的快照与前端时间线展示一致
  - 继续把“节点重放”从内容阶段扩展到更多节点，或补更细粒度的回放保护策略

### 阶段 4：任务记忆架构升级

- 当前状态：`未开始`
- 目标：让记忆层从“摘要展示”升级成“可被 Agent 利用的上下文资产”。
- 计划内容：
  - 用户偏好记忆
  - 失败模式记忆
  - 重试策略记忆
  - 节点产物沉淀与复用

### 阶段 5：评测 / 回归 / 运维能力

- 当前状态：`未开始`
- 目标：让每次升级都能被验证，而不是只靠体感判断。
- 计划内容：
  - Prompt 版本对比
  - 节点成功率统计
  - 失败热点分析
  - 配置变更后的最小回归验证
